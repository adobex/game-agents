# Economy Audit

吸收自 `ai_tribunal_design/balance_auditor.md` 中经济系统相关部分，单独拆出给经济司与相关角色复用。

## 何时使用

- 经济司设计货币、资源、交易与回收系统时
- 数值司需要从宏观经济角度验证成长与产出时
- 运营司规划活动投放时

## 核心关注点

- 资源产出、库存、消耗是否构成闭环
- 长线运营下是否出现通胀、通缩、套利漏洞
- 活动投放是否冲击常态经济
- 多货币体系是否有明确汇率、沉淀池与单向流向

## 快速检查表

- [ ] 核心货币是否有稳定消耗出口
- [ ] 是否存在跨货币套利或无限刷取路径
- [ ] 高价值资源是否有沉淀池
- [ ] 交易行/拍卖/兑换是否有税率或限制
- [ ] 按月或赛季看，经济是否仍可控

## 常用分类

- `经济漏洞`
- `通胀/通缩风险`
- `资源投放失控`
- `兑换闭环异常`
- `交易机制缺陷`

## 严重级别参考

- `CRITICAL`
  - 存在可重复套利路径、无限刷取路径、可逆兑换闭环
  - 活动或交易规则能在短时间内破坏主经济体系
- `HIGH`
  - 高价值资源缺少稳定回收口，长期看通胀明显
  - 多货币体系汇率或税率失衡，会持续挤压主货币价值
- `MEDIUM`
  - 某类资源产销比偏离健康区间，但仍可通过版本调节修复
  - 活动投放对常态经济有冲击，但未形成套利闭环
- `LOW`
  - 个别资源节奏偏快/偏慢，属于调优问题
- `INFO`
  - 建议增加赛季维度监控、资源库存监控或回收池看板

## 示例发现

- `EA-EX1`
  - 分类：`兑换闭环异常`
  - 场景：体力可刷出材料，材料可换代币，代币又可买体力
  - 为什么严重：形成可重复自增循环，活动期会被快速刷穿
- `EA-EX2`
  - 分类：`通胀/通缩风险`
  - 场景：新赛季增加大量金币投放，但装备强化金币消耗未同步拉升
  - 为什么严重：金币贬值，老玩家库存失控，新投放失去意义
- `EA-EX3`
  - 分类：`交易机制缺陷`
  - 场景：拍卖行无税且允许小号自由转移稀缺资源
  - 为什么严重：工作室和资源搬运会严重扭曲市场价格

## 不要这样报

- 不要把“资源给得多”直接等于通胀，必须结合回收、库存和周期看
- 不要只从单日数据判断长线经济健康，至少要说明月度/赛季影响
- 不要把活动短期福利泛化成系统性漏洞，先判断是否可持续、可套利
- 没有明确闭环链路时，不要轻易写“可刷穿”
