# Oracle 数值与逻辑推演机制

**目标**: 通过代码沙盒进行暴力模拟（Monte Carlo）或极值计算，防止数值崩盘或经济通胀。

## 使用场景
- 验证抽卡保底机制在 10000 次抽取下的实际出金率期望。
- 验证战斗伤害公式在等级 1 和等级 99 时的缩放是否平滑。
- 推演游戏中产出与消耗在 30 天后的通胀情况。

## 执行流 (Workflow)
1. **公式/模型确认**：明确需要验算的数学公式或概率模型。
2. **编写验证脚本**：在 `/tmp/` 目录下编写 Python 模拟脚本（如 `simulate_gacha.py`）。
   - 脚本必须包含多次迭代（如 `for _ in range(10000)`）以获取大样本期望。
3. **执行与提取**：运行脚本并捕捉输出结果。
4. **结论反哺**：根据数据结果，调整策划案中的具体数值填列，并在文档中附上“Oracle 模拟证明”。
