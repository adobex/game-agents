# Monetization Audit

吸收自 `ai_tribunal_design/balance_auditor.md` 中“付费公平性 + 概率系统”相关部分，供付费司、经济司、数值司复用。

## 何时使用

- 付费司设计首充、礼包、通行证、抽卡时
- 经济司检查商业化是否冲击经济闭环时
- 数值司验证付费成长与免费成长差距时

## 核心关注点

- 付费是否只是加速，还是直接突破上限
- 抽卡概率、保底、最坏情况消耗是否透明合理
- 商业化是否破坏 PvP/PvE 公平和长期留存
- 是否存在隐性付费墙、概率陷阱或过度负面舆情风险

## 快速检查表

- [ ] 是否存在不可弥合的付费碾压
- [ ] SSR/稀有物的概率、软硬保底是否明示
- [ ] 免费玩家是否有长期但可达的追赶路径
- [ ] 商业化活动是否破坏核心玩法节奏
- [ ] 用户心理预期与实际收益是否匹配

## 常用分类

- `付费碾压`
- `概率设计缺陷`
- `保底机制异常`
- `隐性付费墙`
- `商业化包装风险`

## 严重级别参考

- `CRITICAL`
  - 付费内容直接突破公平边界，形成不可弥合的 PvP/PvE 优势
  - 抽卡概率、保底或实际收益与宣称严重不符
- `HIGH`
  - 首充、礼包、通行证提供超规格战力，压缩免费成长路径
  - 商业化节奏明显破坏核心玩法和长期留存
- `MEDIUM`
  - 付费收益偏高或包装存在误导，但尚未形成硬付费墙
  - 免费追赶路径存在，但周期过长、体验过差
- `LOW`
  - 包装文案、档位梯度、展示顺序仍可优化
- `INFO`
  - 建议补充披露文案、A/B 测试点或舆情观察项

## 示例发现

- `MR-EX1`
  - 分类：`隐性付费墙`
  - 场景：核心养成材料几乎只在礼包和高价通行证产出
  - 为什么严重：免费玩家长期无法追平，系统会被感知成“不给钱别玩”
- `MR-EX2`
  - 分类：`保底机制异常`
  - 场景：软保底概率提升与硬保底继承关系写法冲突
  - 为什么严重：玩家预期和真实消耗不一致，极易引发信任问题
- `MR-EX3`
  - 分类：`商业化包装风险`
  - 场景：限时礼包文案暗示“必得稀有角色”，实际只给碎片
  - 为什么严重：即使数值没炸，也会造成高舆情和投诉风险

## 不要这样报

- 不要把“玩家愿意买”当作没问题，仍要检查公平、透明与长期影响
- 不要把一切付费加速都报成 `付费碾压`，先判断是否突破上限和竞争公平
- 不要只盯概率，不看免费追赶路径、节奏破坏和认知预期
- 没有玩家价值对照时，不要泛泛写“商业化太重”
