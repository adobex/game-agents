# =========================================================
# PROJECT GOALS
# =========================================================
- 生成竖屏触屏手游形式的高质量 H5 游戏  
- 必须可在 file:// 下直接运行  
- 工程结构清晰、可扩展、可维护

# =========================================================
# 1. TECH STACK
# =========================================================
必须使用：HTML5 Canvas + 原生 JS(ES6+) + CSS3  
禁止 import/export、禁止第三方库、禁止构建工具、禁止跨域加载  
所有 JS 必须通过 <script> 顺序加载并使用命名空间或 IIFE。


# =========================================================
# 2. DIRECTORY STRUCTURE
# =========================================================
root/
 index.html
 css/style.css
 js/config.js   ← 必须第一个加载
 js/utils.js
 js/loader.js
 js/input.js
 js/main.js
 assets/images/
 assets/audio/

禁止深层目录；禁止 "../"；禁止网络资源。


# =========================================================
# 3. SCRIPT LOAD ORDER
# =========================================================
config.js → utils.js → loader.js → input.js →（所有新增模块）→ main.js  
禁止调整顺序，禁止放模块到 config.js 前。


# =========================================================
# 4. DATA RULES
# =========================================================
所有数值、颜色、速度、尺寸、文本内容必须写入 config.js。  
禁止 Magic Number。


# =========================================================
# 5. UI BASE RULES
# =========================================================
必须使用以下固定 DOM 结构：

<div id="game-container">
 <canvas id="game"></canvas>

</div>

# =========================================================
# 6. SCREEN RULES
# =========================================================
逻辑分辨率固定：720×1280（竖屏）  
Canvas 必须 letterbox；禁止拉伸  
禁止使用 window.innerWidth/innerHeight 定位 UI。


# =========================================================
# 7. HARD LIMIT: 9:16
# =========================================================
所有绘制/UI/输入判定必须在 (0,0)-(720,1280) 内  
DOM 必须 absolute 且相对 #game-container，禁止相对 body/window。


# =========================================================
# 8. CODE STYLE
# =========================================================
注释使用简体中文；使用 const/let；必须模块化；禁止 main.js 过度堆逻辑。


# =========================================================
# 9. ANIMATION
# =========================================================
使用 deltaTime；公共动画用 IIFE；提示动画 ≥0.8 秒；禁止 2020+ emoji。

# =========================================================
# 18. MOBILE CONTROL RULES（竖屏手游操作规范）
# =========================================================

# ---- 18.1 虚拟摇杆 ----
默认隐藏；仅在下半屏触摸出现（y>640）  
触摸点为摇杆中心；松开消失  
输出 dx,dy,angle  
属于 layer-ui-float  
尺寸由 CONFIG 配置  
必须支持鼠标 + 触屏

# ---- 18.2 触控区域划分 ----
下半屏 → 移动（摇杆）  
上半屏 → 技能按钮  
两者不得冲突。

# ---- 18.3 战斗按钮 ----
攻击按钮 → 右下且不进入摇杆区  
技能按钮 → 上半区  
必须 pointer-events:auto

# ---- 18.4 操作适配 ----
摇杆 → move(dx,dy)  
单击 → attack()  
长按 → block()  
移动与攻击必须并行  
3D 游戏镜头必须自动化（Auto Camera）

# ---- 18.5 操作验证 ----
摇杆出现正常；按钮不冲突；移动+攻击并行；区域检测准确。


# =========================================================
# 19. GAMEPLAY INTENT ANALYSIS（玩法意图系统）
# =========================================================

# ---- 19.1 横屏需求 → 自动适配竖屏 ----
依赖宽视野 → 自动压缩视野 + 居中视图  
输出提示：已适配竖屏。

# ---- 19.2 自由镜头 → 自动镜头 ----
若需要自由镜头 → 自动替换 Auto Camera（禁止手动旋转镜头）

# ---- 19.3 是否需要摇杆 ----
需要摇杆：动作、探索、地牢、生存、移动射击  
禁止摇杆：经营、放置、卡牌、解谜、三消、视觉小说  
若玩法无需移动 → 禁止生成摇杆

# ---- 19.4 自动识别玩法 ----
包含“移动/探索/走位/追逐” → 摇杆  
包含“建造/经营/拖拽/放置” → 点击  
仅“攻击技能”无移动 → 按钮  
纪念碑谷类 → 点击，不得生成摇杆

# ---- 19.5 输入冲突自动修正 ----
若玩法描述出现自由镜头 + 摇杆 → 使用：摇杆 + 自动镜头

# ---- 19.6 输出操作选择 ----
Cursor 必须输出选择的操作方式并解释原因。

# 20. PERFORMANCE BUDGET（性能预算）
# =========================================================
目标帧率：60 FPS；低于 45 FPS 必须优化  
单帧 Canvas 绘制调用 ≤ 300  
粒子数量 ≤ 100（普通） / 300（技能）  
3D 场景 mesh 总数 ≤ 150  
贴图最大尺寸 ≤ 1024×1024  
禁止实时阴影（仅允许模拟阴影）  
禁止复杂 Bézier 反复绘制


# =========================================================
# 21. ASSET SIZE BUDGET（资源体积预算）
# =========================================================
单图 ≤ 200 KB（非主角 ≤ 100 KB）  
优先 WebP；次选压缩 PNG  
音频 ≤ 200 KB（mp3）  
初版项目体积 ≤ 3 MB  
loader.js 必须输出加载占比报告
