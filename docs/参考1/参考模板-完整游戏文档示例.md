## 3. 客户端和服务器逻辑

### 3.1 客户端逻辑

客户端负责以下功能：

**显示和交互：**
- 渲染功能界面的所有UI元素
- 展示玩家当前状态（等级、资源、进度等）
- 处理玩家的点击、拖拽等输入操作
- 播放动画效果和音效反馈

**本地验证：**
- 检查玩家操作是否合法（例如：是否有足够资源）
- 验证输入参数的有效性
- 进行基础的逻辑判断（例如：是否满足操作条件）

**数据缓存：**
- 缓存从服务器获取的配置数据
- 保存玩家的操作记录（用于断线重连）
- 临时存储未提交的操作状态

**表现层逻辑：**
- 根据数据更新UI显示
- 计算和展示动画过渡效果
- 管理界面元素的显隐和交互状态

### 3.2 服务器逻辑

服务器负责以下功能：

**数据持久化：**
- 存储玩家在该功能中的所有进度和状态
- 记录玩家的历史操作和成就
- 保存功能相关的全局数据（如排行榜）

**业务逻辑验证：**
- 验证客户端请求的合法性
- 检查操作是否符合游戏规则
- 防止作弊和异常操作
- 验证资源消耗和奖励计算的正确性

**核心计算：**
- 计算玩家操作的结果（战斗结算、任务完成度等）
- 处理随机事件和概率判定
- 计算奖励数量和类型
- 更新玩家的各项数值和状态

**数据同步：**
- 将更新后的数据发送给客户端
- 处理多客户端之间的数据一致性
- 管理实时数据的推送和广播

**系统联动：**
- 触发其他系统的相关逻辑（如成就系统）
- 更新全局数据（如排行榜、统计数据）
- 发送系统通知和奖励邮件

### 3.3 客户端-服务器交互

**协议1：进入功能**
- 触发时机：玩家点击进入功能时
- 客户端请求：发送玩家ID，请求功能数据
- 服务器响应：返回玩家当前状态、可用资源、配置数据等
- 数据内容：
  - 玩家等级、经验值
  - 当前拥有的资源数量
  - 功能的当前进度
  - 可用的选项和操作

**协议2：执行核心操作**
- 触发时机：玩家执行主要玩法操作时
- 客户端请求：发送操作类型、操作参数
- 服务器响应：验证操作，计算结果，返回操作结果
- 数据内容：
  - 操作是否成功
  - 消耗的资源
  - 获得的奖励
  - 更新后的状态

**协议3：领取奖励**
- 触发时机：玩家完成任务或达成条件后
- 客户端请求：发送奖励ID或任务ID
- 服务器响应：验证条件，发放奖励，返回奖励详情
- 数据内容：
  - 奖励物品列表
  - 更新后的背包和资源
  - 成就或称号解锁信息

**协议4：同步数据**
- 触发时机：定期或在关键节点
- 客户端请求：请求最新数据
- 服务器响应：返回当前最新的状态数据
- 数据内容：
  - 全局状态（排行榜、活动进度等）
  - 玩家最新数据（防止数据不一致）

**协议5：退出功能**
- 触发时机：玩家退出功能时
- 客户端请求：发送退出请求，上传未提交的数据
- 服务器响应：保存玩家状态，返回确认信息
- 数据内容：
  - 最终状态确认
  - 结算信息（如有）

| reset_time | string | 重置时间（每日5点） | "05:00:00" |

## 5. 其他说明

### 5.1 特殊情况处理

**断线重连：**
- 客户端重连后，向服务器请求当前状态
- 服务器返回玩家最后保存的状态
- 如果有未完成的操作，根据情况恢复或作废

**资源不足：**
- 客户端提前检查并给出提示
- 服务器再次验证，不足则返回错误码
- 引导玩家前往获取资源的途径

**作弊检测：**
- 服务器记录操作时间戳，检测异常频率
- 验证操作的先后顺序和逻辑合理性
- 对异常行为进行标记和处理

### 5.2 数据统计需求

需要统计的数据指标：
- 功能的日活跃人数和次日留存
- 玩家平均停留时间和操作次数
- 各个关卡/任务的完成率
- 资源消耗和产出的平衡数据
- 付费转化率（如果有付费点）
