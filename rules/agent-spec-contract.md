# Agent Spec Contract

用于约束 `agents/*/SOUL.md` 的最小可执行规格，避免再次出现只有模板、没有契约的 Agent 定义。

## 必填项

每个 Agent 的 `SOUL.md` 至少要明确以下内容：

- `目标`：一句话说明该 Agent 解决什么问题
- `输入`：上游是谁、会带来什么上下文、最小输入包是什么
- `输出`：交付什么文档、结论或任务包，交给谁
- `标准交付物`：至少 2-3 项可检查的产物
- `Skills`：只允许引用真实存在的 skill 文件
- `工作流与交互机制`：必须写成该 Agent 自己的流转步骤，禁止保留通用占位模板

## 禁止项

- 禁止使用 `待细化`、`待补充`、`TBD` 作为长期保留内容
- 禁止把能力描述写进 `Skills`，`Skills` 只保留真实文件名
- 禁止在输入阶段要求一次性加载所有上下文，必须遵守渐进式披露

## 建议写法

- 先写最小输入包，再写扩展上下文
- 先写标准输出，再写可选附录
- 优先描述“什么时候交付给谁”，而不是空泛描述“负责什么”
