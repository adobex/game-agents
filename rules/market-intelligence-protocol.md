# Market Intelligence Protocol

用于约束 `analyst` 及其下游角色在处理市场情报、发行复盘和竞品营销结论时的证据分级与转译方式。

## 1. 证据分级

所有关键判断尽量标注以下证据等级之一：

- `A - 直接观察`
  - 可从公开页面、广告素材库、商店页、榜单、截图、视频、新闻稿中直接看到
  - 例：Icon 样式、截图文案、素材剧情、节点时间、榜单峰值
- `B - 高可信推断`
  - 由多个公开信号交叉支持，但不是后台直读数据
  - 例：渠道侧重、买量节奏、出价模型偏好、节日投放节点
- `C - 方向性假设`
  - 有一定行业规律支持，但缺少足够公开证据
  - 例：具体预算比例、精确 ROI、特定后台优化事件

## 2. 表达规则

- `A` 可直接写成“观察到 / 页面显示 / 素材体现”
- `B` 必须写成“推测 / 判断 / 高概率是”
- `C` 必须写成“可能 / 倾向 / 假设”，且不要作为核心结论唯一支撑
- 若关键结论只有 `C` 级证据，不得写成确定判断

## 3. 最小证据台账

《发行复盘报告》至少应包含：

- 关键判断
- 证据等级
- 主要依据
- 不确定点

推荐表头：

```markdown
| 结论 | 证据等级 | 主要依据 | 不确定点 |
|------|----------|----------|----------|
```

## 4. 下游转译等级

`analyst` 的结论进入下游前，必须再标记“如何使用”：

- `可直接采用`
  - 可直接转成我方输入约束或设计前提
- `需本地化改写`
  - 思路可借鉴，但必须按我方产品、市场、资源条件改写
- `仅作参考`
  - 只作为情报背景，不直接进入方案

## 5. 下游消费规则

### 给 `positioning`

- 优先消费：受众标签、视觉语言、转化标签、差异化切口
- `A/B` 可进入定位论据
- `C` 只能作为待验证假设，不直接写成 USP

### 给 `liveops`

- 优先消费：节日节点、投放节奏、热点窗口、素材节点打法
- 默认 `需本地化改写`
- 不得直接照搬竞品活动日历

### 给 `monetization`

- 优先消费：素材中的付费钩子、付费包装方式、抽卡叙事、礼包心理预期
- 默认 `需本地化改写`
- 不得把竞品价格带、保底或付费强度直接抄成我方案

## 6. 反模式

- 把 `C` 级假设写成确定事实
- 用单一素材样本推导完整发行策略
- 把竞品发行动作直接抄成我方执行方案
- 不做本地化条件判断，就把情报结论下发给下游执行层

## 7. HTML 输出偏好

当 `analyst` 需要输出 HTML / 演示型报告时，默认参考：

- `docs/参考3/织梦森林—原始页面.html`

优先学习并复用以下层面：

- 页面级信息架构：Hero、章节导航、分段叙事、结论收束
- 视觉表达：深色基底、发光卡片、重点数字、表格、标签体系
- 交互方式：粘性导航、滚动分节、图表区块、渐进式展开感
- 信息表达手法：先结论后展开、图表与文字并置、关键信号高亮

注意：

- 优先参考其排版、视觉语言和交互方式，不等于机械复制内容
- 若任务明确要求“原样下载”或“原样保存网页”，则直接保存源码，不做重排
